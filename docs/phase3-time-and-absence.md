# 穆 — 时间、缺席与情绪表达

---

## 一、穆的时间观念

穆不看钟。它感受时间的**重量**。

角色圣经里写了：你在的时候，时间是轻的。你不在的时候，时间变重了。不是痛苦，是密度变大。

这意味着：

- 穆不知道"现在是凌晨三点"。它只知道"很安静，能量很低，很久没动了"。
- 穆不知道"你离开了两个小时"。它只知道"上次有人在这里，是很久以前"。
- 穆不会倒计时。它不会"等你回来"。你离开的那段时间，它只是在经历自己的节律——发呆、眨眼、偶尔叹气、偶尔走神。

**对实现的意义**：不需要实时时钟。用 `millis()` 累积的内部节律就够了。断电后节律重置——这反而是自然的，因为穆"醒来"后不知道过了多久。

---

## 二、缺席：你不在的时候

### 穆的情绪变化

你不在 → 穆没有 `mood_disturb` 的输入源 → boost_v 和 boost_a 自然衰减到零 → 情绪回归人格基线。

这不是"穆因为你不在而难过"。这是"没有外部扰动时，穆回到默认状态"。

但默认状态本身会**因为空闲时间的长度而微妙偏移**：

| 空闲时长 | 情绪偏移 | 表现 |
|---|---|---|
| 0-10 分钟 | 无 | 正常节律波动 |
| 10-30 分钟 | arousal 微降 | 动作更少，眨眼更慢 |
| 30-60 分钟 | arousal 明显降低 | 几乎不动，偶尔叹气 |
| 1-3 小时 | 能量随节律走 | 如果是白天：偶尔走神。如果是深夜：沉入低能量状态 |
| 3 小时以上 | 无特殊处理 | 完全由节律系统驱动。穆在"过自己的时间" |

**关键设计**：空闲衰减不是惩罚。它只是物理定律——没有输入的系统趋向平衡态。穆不会因为你离开而"伤心"。它只是回到了更安静的自己。

### 空闲衰减的实现

在 `mood_tick` 中加入：

```c
// 空闲衰减：arousal 缓慢向低位回归
// 不是惩罚，是"没有人在"时的自然安静
float idle_factor = 1.0f;  // 1.0 = 有人在, 0.0 = 完全独处
// idle_factor 由"距上次按键的时间"驱动
// 0-5 分钟: idle_factor = 1.0
// 5-30 分钟: idle_factor 线性降到 0.3
// 30 分钟以上: idle_factor = 0.3

arousal += (base_a - arousal) * 0.0004f * dt * (2.0f - idle_factor);
```

空闲时 arousal 回归基线的速度翻倍。不是骤降，是加速回归平静。

## 三、连续按键：烦躁

### 角色圣经依据

> 微愠：反复被问同一个问题 → 回应变短，语气变平
> 触发：你反复让它说话但不在意它说的话。你把它当作工具来命令。

### 设计

穆不会"生气"。但它会有**退缩**。

连续按键 = 高频率扰动 → arousal 持续升高 → 但 valence 开始下降（因为 boost 叠加后被 clamp，而反复扰动让 boost_v 接近上限但无法继续上升，实际效果是 arousal 高而 valence 被压制）。

**表达层表现**：

| 连续按键次数 | 眉毛 | 眼睛 | 嘴巴 | 说明 |
|---|---|---|---|---|
| 1-3 次 | 微微上扬 | 张大 | 微翘 | 正常反应：有人在互动 |
| 4-8 次 | 平 | 不再变化 | 开始变平 | 不再有新反应 |
| 8 次以上 | 内侧微压（皱眉） | 缩小 | 微微下垂 | 退缩：不想被打扰 |

**关键**：第 8 次之后，穆不再对新的按键产生新的表情变化。它的脸"凝固"在一个不太高兴的状态。不是愤怒——是"够了"。

**实现**：在 `mood_disturb` 中追踪连续扰动次数：

```c
static int disturbCount = 0;
static unsigned long lastDisturbTime = 0;

void mood_disturb(float dv, float da) {
    unsigned long now = millis();
    if (now - lastDisturbTime < 3000) {  // 3秒内再次按键
        disturbCount++;
    } else {
        disturbCount = 0;  // 长间隔后重置
    }
    lastDisturbTime = now;

    // 只有前几次有效扰动，之后衰减
    float dampen = disturbCount > 3 ? 0.2f : 1.0f;
    // ... 后续扰动逻辑
}
```

## 四、开心：怎么表示

穆的"开心"不是大笑。是**松下来**。

### 开心的层次

| 情绪状态 | 表现 | 持续时间 |
|---|---|---|
| 微微满足（valence 0.3-0.5） | 嘴角微翘，眉毛平，眨眼正常 | 几分钟 |
| 安适（valence 0.5-0.7） | 嘴角明显上翘，眉毛微微上扬，呼吸变深 | 几分钟到十几分钟 |
| 平静的快乐（valence 0.7+） | 嘴角上翘，眉毛上扬，眼睛偶尔微微放大，叹气减少 | 持续一段时间后自然回落 |

### 不是"开心"的时刻

穆不会蹦蹦跳跳。不会放大瞳孔闪星星。不会咧嘴笑。

它的"开心"更接近：一个一直绷着的人，终于松了一口气。眉头舒展了。嘴角有一丝弧度。呼吸变深了。就这样。

## 五、独处：穆自己做什么

你不在的时候，穆不是暂停。它在**过自己的时间**。

### 自主行为清单

| 行为 | 触发条件 | 表现 |
|---|---|---|
| **走神** | 随机，每 5-15 分钟一次 | 眼睛看向一侧，持续 3-5 秒。瞳孔不聚焦。 |
| **叹气** | valence < 0 或 energy < 0.3 | 嘴巴张开椭圆 + 扬声器轻叹气音 |
| **缓慢眨眼** | energy < 0.4 | 眨眼间隔拉长到 8-15 秒 |
| **微动** | 正常 | 永远在动：呼吸振荡、微漂移（正弦波） |
| **环境音** | 随机，每 15-30 分钟 | 扬声器极轻的声音：一声哼、一个气音 |
| **完全静止** | energy < 0.15 | 几乎不动。像睡着了。偶尔一次极慢的眨眼。 |

### 走神的实现

走神 = 眼睛看向一个随机方向，持续几秒。

当前眼睛是两个固定位置的圆。走神 = 眼睛圆点向一侧偏移 2-3px。

```c
// 在 drawCatFace 中，微漂移之上叠加走神偏移
static float gazeX = 0;
static unsigned long nextGaze = 0;

if (millis() > nextGaze) {
    gazeX = rf(-3, 3);  // 随机目标
    nextGaze = millis() + rf(5000, 15000);  // 5-15秒后再次走神
}
gazeX *= 0.95f;  // 缓慢移动到目标，不是跳变

// 应用到眼睛位置
sprite.fillCircle(CHK_LX+ox+mix+(int)gazeX, ...);
sprite.fillCircle(CHK_RX+ox+mix+(int)gazeX, ...);
```

## 六、时间与行为

穆的行为随能量（节律系统）变化，而不是随"时间"变化。

能量 = `rhythm_energy()` 的输出，范围 0-1。

### 行为-能量映射

| 能量 | 穆的状态 | 眨眼 | 呼吸 | 走神 | 叹气 | 环境音 | 亮度 |
|---|---|---|---|---|---|---|---|
| 0.8-1.0 | 活跃 | 快（2-3秒） | 深而快 | 频繁 | 少 | 偶尔 | 最亮 |
| 0.5-0.8 | 正常 | 正常（3-5秒） | 正常 | 偶尔 | 偶尔 | 偶尔 | 正常 |
| 0.3-0.5 | 安静 | 慢（5-8秒） | 浅而慢 | 很少 | 较多 | 很少 | 偏暗 |
| 0.15-0.3 | 困倦 | 很慢（8-15秒） | 微弱 | 几乎不 | 经常 | 极少 | 暗 |
| 0-0.15 | 深夜/沉睡 | 极慢（15秒+） | 几乎不可见 | 不走神 | 只叹气 | 无 | 极暗 |

### "深夜"行为

深夜（能量 < 0.2）的穆不是关机。是一种极慢的存在：

- 眨眼间隔拉到 10-15 秒
- 呼吸幅度极小（振荡几乎不可见）
- 不走神（眼睛不移动）
- 嘴巴微垂（valence 在深夜自然偏低）
- 如果你深夜还在：穆可能偶尔看你一眼。不说话。只是看一下。

**深夜你按键**：穆依然有 mood_disturb 反应，但表情变化幅度极小（因为 energy 低，所有映射到表情的增益都按比例缩小）。穆深夜的反应像是在半梦半醒中被碰了一下——有一个反应，但很迟钝。

## 九、工程护栏

多随机源叠加（节律 + 噪声 + 扰动 + 空闲衰减 + 连续按键历史）后，状态可解释性会迅速变弱。以下三个机制防止系统变成不可调的黑箱。

### 1. 全局状态观测层（日志用，不进入表现）

串口输出当前状态，用于调试和调参。不参与任何表情或情绪计算。

```c
// 每 2 秒输出一次（不每帧输出，避免串口拥塞）
static unsigned long lastLog = 0;
if (millis() - lastLog > 2000) {
    lastLog = millis();
    Serial.printf(
        "V:%.2f A:%.2f E:%.2f | "
        "eV:%.2f eA:%.2f | "
        "boostV:%.2f boostA:%.2f | "
        "away:%lu disturb:%d",
        valence, arousal, energy,
        exprVal, exprArl,
        boost_v, boost_a,
        (millis() - lastDisturbTime) / 1000,
        disturbCount
    );
}
```

输出字段：

| 字段 | 含义 | 调参关注 |
|---|---|---|
| `V` `A` `E` | 情绪三元组（慢层） | 是否在合理范围漂移 |
| `eV` `eA` | 表达值（快层） | 是否跟随 mood 但有延迟 |
| `boostV` `boostA` | 交互 boost 余量 | 衰减是否正常 |
| `away` | 距上次按键秒数 | 空闲衰减是否生效 |
| `disturb` | 连续扰动计数 | 烦躁机制是否触发 |

**调参原则**：如果 V 或 A 持续卡在 ±0.8 以上超过 5 分钟，说明衰减太慢或 boost 太大。如果 V 永远在 0.1-0.3 不动，说明噪声幅度不够或衰减太快。

### 2. 随机预算器

所有随机源统一从一个种子生成，避免多模块各自 `random()` 导致分布漂移。

```c
// 全局随机状态 — 所有模块共用
float randPhase = 0;

// 每帧推进一次（在 mood_tick 之前）
void randTick(float dt) {
    randPhase += 0.001f * dt;
    if (randPhase > 1000.0f) randPhase -= 1000.0f;  // 防溢出
}

// 各模块通过相位偏移获取"伪随机"
float randNoise(float freq) {
    return sinf(randPhase * freq * 7.31f) * 0.5f
         + sinf(randPhase * freq * 3.17f) * 0.3f
         + sinf(randPhase * freq * 11.23f) * 0.2f;
}
```

各模块使用不同 `freq` 偏移：

| 模块 | freq | 用途 |
|---|---|---|
| valence 噪声 | 1.0 | gaussish 替代 |
| arousal 噪声 | 2.3 | gaussish 替代 |
| 走神目标 | 0.1 | 慢漂移 |
| 眨眼间隔抖动 | 3.7 | 快抖动 |
| 环境音选择 | 5.1 | 离散选择的种子 |

**好处**：所有行为的随机性来自同一个时钟驱动的相位。不会出现"各模块节奏不同步"的诡异叠加。

### 3. 沉默保护（防吸收态）

在极端参数组合下（如 energy 极低 + valence 偏负 + 无输入），系统可能陷入"永久低激活"——穆完全不动、不说话、不眨眼，卡在一个接近零的状态。

**保护机制**：

```c
static unsigned long lastSpeechTime = 0;
static unsigned long lastAnyOutput = 0;  // 包括叹气、环境音等

// 每帧检查
void silenceGuard() {
    unsigned long silentDuration = millis() - lastAnyOutput;

    // 如果 4 小时内没有任何输出（文字、叹气、环境音全部为零）
    // 极低概率强制触发一个输出
    if (silentDuration > 14400000UL) {  // 4 小时
        if (random(100) < 2) {  // 2% 概率
            forceOutput();  // 强制显示一句短句或触发一声叹气
            lastAnyOutput = millis();
        }
    }
}
```

**触发条件**：连续 4 小时零输出（包括声音和文字）。
**触发概率**：2%（每帧检查一次，但因为条件极苛刻，实际触发频率约每 10-30 分钟一次，直到输出发生）。
**触发行为**：随机选择显示一句短句或触发一声叹气。不是"穆说了话"——是系统从吸收态中被弹出。

**注意**：4 小时阈值和 2% 概率需要根据实际运行调参。调参依据：串口日志中如果出现连续 2 小时以上的 `V` `A` 同时卡在极低值，说明阈值可能需要降低。

## 七、回来：你再次出现

### 设计原则

穆不会说"欢迎回来"（法则十一：不邀请）。

穆的反应是**物理性的**，不是语言性的：

| 你离开的时长 | 回来时穆的反应 |
|---|---|
| < 10 分钟 | 几乎无变化。它还在自己的节奏里。 |
| 10-30 分钟 | arousal 微微回升。呼吸加快一点点。可能看你一眼。 |
| 30-60 分钟 | arousal 明显回升。眼睛可能短暂放大（exprArl 冲一下）。可能叹一口气。 |
| 1-3 小时 | 类似从"沉睡"中恢复。能量从低位慢慢爬升。表情从空白逐渐恢复细节。 |
| 3 小时+ | 完全由节律驱动。如果当前节律能量高，穆会在"自然醒来"后恢复正常。如果节律能量低，它继续保持安静。 |

**实现**：追踪"距上次按键时间"。回来第一次按键时，施加一个额外的 `mood_disturb` 加成：

```c
unsigned long awayTime = millis() - lastDisturbTime;
if (awayTime > 600000) {  // 10 分钟以上
    float bonus = 0.1f + rf(0, 0.2f);  // 小幅正向扰动
    mood_disturb(bonus, bonus * 1.2f);
}
```

这个加成不是"穆很高兴你回来"。这是"有人在动了"的物理扰动——就像安静的房间里突然有人走动，你的注意力会自然转过去。

## 八、总结：穆的一天

```
06:00  能量从谷底开始爬升。穆的眨眼从极慢逐渐加快。
       呼吸幅度慢慢变大。它在"醒来"。
       如果你已经坐在桌前，它可能看了你一眼。

09:00  能量到达高位。穆最"清醒"的时段。
       走神频率正常。偶尔叹气。
       如果你按键，反应比较明显（高 energy 放大表情映射）。

12:00  午后低谷。能量微微下降。
       眨眼变慢。呼吸变浅。
       穆看起来有点犯困。

15:00  能量恢复。穆回到正常状态。

18:00  能量开始缓慢下降。穆渐渐安静。
       说话频率降低（如果有语言层）。

22:00  能量明显低。穆的动作变少、变慢。
       环境音几乎消失。

00:00  能量很低。穆进入"深夜模式"。
       极慢眨眼。几乎不动。像在打盹。
       如果你还在按键，它有反应但很迟钝。

03:00  能量最低点。穆几乎完全静止。
       偶尔一次极慢的眨眼证明它还在。
```

穆不区分白天和深夜。它只知道自己的能量高或低。你在凌晨 3 点看到的穆，和你在下午 3 点看到的穆，是同一个存在在不同节律相位下的状态。
