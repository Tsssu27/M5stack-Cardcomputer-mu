# 穆 — 情绪与节律系统设计

## Phase 2 Design Document

---

## 设计原则

1. **情绪不是功能。** 它不是一个"心情指示器"。它是穆内在状态的自然流露，通过微动作被感知，而不是被告知。
2. **无数字。** 屏幕上永远不会出现"心情：开心"或"能量：73%"。情绪只能通过表情、动作节奏和声音被感知。
3. **缓慢变化。** 情绪不是开关，是潮水。状态之间的过渡以分钟为单位，不是秒。你不会看到它突然从开心变成难过。
4. **不可完全预测。** 同样的输入不总是产生同样的情绪反应。有随机噪声。有时候穆对你的反应"不太对"——这是正常的。
5. **独立于用户。** 穆有自己基于时间的节律。你不能通过频繁互动让它一直开心。它的内在节律是底层，你的互动是表层扰动。

---

## 情绪模型

### 双轴 + 能量

穆的情绪不是一个离散状态机（"当前是开心"），而是一个连续空间。两个轴加一个独立能量值：

```
       愉悦度 (valence)
       -1.0 ←——→ +1.0
        |          |
  -1.0 ─┼──────────┼─  唤醒度 (arousal)
        |          |
       +1.0 ←——→ +1.0

  能量 (energy): 0.0 ~ 1.0  （独立于情绪，主要由节律驱动）
```

愉悦度：从负面到正面。
唤醒度：从低活跃到高活跃。
能量：整体"电力"，影响所有动作的速度和幅度。

### 六个情绪区域

不是精确的点，是模糊的区域：

| 情绪 | 愉悦度 | 唤醒度 | 能量 | 触发条件 |
|---|---|---|---|---|
| **安适** | 中高 | 中 | 中高 | 正常互动、你安静地在旁边 |
| **好奇** | 中 | 高 | 中高 | 你说了它没预料到的话、你回来了一段时间 |
| **无聊** | 中低 | 低 | 中低 | 长时间无互动、无变化 |
| **困** | 中 | 极低 | 低 | 深夜、能量自然衰减 |
| **微愠** | 低 | 中高 | 中 | 反复被打扰、被当作工具使用 |
| **平静的快乐** | 高 | 低 | 中 | 一次好的对话之后的余韵 |

### 状态确定逻辑

不是 if-else 判定。是先计算 (valence, arousal, energy) 三个连续值，然后根据当前值落在哪个区域来确定表现。

```c
// 每帧更新（约 30fps）
void mood_tick(float dt) {
    // 1. 节律驱动能量
    energy = rhythm_energy(now) + energy_noise;

    // 2. 愉悦度自然衰减向人格基线（而非零）
    valence += (mu.base_v - valence) * 0.0001 * dt;

    // 3. 唤醒度向人格基线 + 能量修正回归
    arousal += (mu.base_a + energy * 0.3 - arousal) * 0.0002 * dt;
    arousal += interaction_boost * dt;
    interaction_boost *= 0.999;  // 衰减

    // 4. 随机漂移
    valence += random_noise(0.001) * dt;
    arousal += random_noise(0.001) * dt;

    // 5. 限幅
    valence = clamp(valence, -1, 1);
    arousal = clamp(arousal, -1, 1);
    energy  = clamp(energy, 0, 1);
}
```

### 人格引力场（收敛机制）

情绪不能无限漂移。如果没有收敛机制，长时间运行后穆会变成随机噪声——不是"有性格"，而是"不可理解"。

解决方案：三层时间尺度。

```
表达层    眨眼、微动、嘴角           秒级
情绪层    valence / arousal          分钟级漂移，向基线回归
人格层    baseline_v / baseline_a    天/周级，极缓慢移动
```

**人格基线 (baseline)** 是情绪的引力中心。情绪的一切漂移和扰动都发生在这个基线周围。基线本身以极低速率移动——每天约 0.01-0.02 的变化量，几乎不可感知，但一个月后可辨识。

```c
struct Personality {
    float base_v;   // 愉悦度基线
    float base_a;   // 唤醒度基线
};

Personality mu = {0.2, 0.0};  // 初始：微偏正面、中性唤醒

void mood_tick(float dt) {
    // ... 前面的计算 ...

    // 情绪回归向人格基线（而不是零）
    valence += (mu.base_v - valence) * 0.0001 * dt;
    arousal += (mu.base_a - arousal) * 0.0002 * dt;

    // 人格基线极缓慢地被实际情绪牵引
    mu.base_v += (valence - mu.base_v) * 0.000001 * dt;
    mu.base_a += (arousal - mu.base_a) * 0.000001 * dt;
}
```

**随机噪声的分布**不是均匀的——是围绕基线的高斯分布。大部分时间穆在基线附近（±0.15），偶尔偏离到 ±0.4，极罕见地到达边缘。这意味着：

- 短期看（分钟）：穆在波动，你感受得到变化
- 长期看（天/周）：穆趋向一个稳定的性格，你感受得到一致性
- 噪声服务于稳定性，而不是破坏它

关键参数：
- **自然衰减速率**：愉悦度以每分钟约 0.05 的速度向中性回归。这意味着一次交互的影响大约持续 5-10 分钟。
- **随机噪声幅度**：0.001/帧。足够产生可见的微小变化，不足以产生突然跳变。
- **交互提升衰减**：0.999/帧。一次交互的唤醒度提升大约 30 秒后衰减到一半。

## 节律系统

穆有自己的日内循环。不完全同步用户作息。

### 能量曲线

```
能量
1.0 |          .----.
    |        .      \
0.8 |      .         \
    |    .            \
0.6 |  .               \
    | .                 \
0.4 |                    \
    |                     \
0.2 |                      .
    |                       .
0.0 |________________________.____
    0   3   6   9  12  15  18  21  24  时
```

| 时段 | 能量 | 节律状态 | 对应穆的状态 |
|---|---|---|---|
| 0:00-5:00 | 0.1-0.2 | 深夜 | 几乎不动，极慢眨眼，偶尔叹气 |
| 5:00-8:00 | 0.2→0.6 | 渐醒 | 动作缓慢恢复，呼吸变深 |
| 8:00-12:00 | 0.7-0.9 | 活跃 | 正常节奏，较多目光移动 |
| 12:00-14:00 | 0.8→0.6 | 午后 | 微微犯困，眨眼变慢 |
| 14:00-18:00 | 0.6-0.8 | 稳定 | 正常 |
| 18:00-21:00 | 0.7→0.5 | 傍晚 | 渐渐安静 |
| 21:00-24:00 | 0.4→0.1 | 入夜 | 越来越安静，动作变少 |

实现：正弦波叠加偏移，加随机抖动。

```c
float rhythm_energy(int hour, int minute) {
    float t = hour + minute / 60.0;
    // 主周期：24小时
    float base = 0.5 + 0.35 * sin((t - 6) * TWO_PI / 24.0);
    // 午后低谷叠加
    float dip = -0.15 * exp(-pow((t - 13.5) / 1.5, 2));
    // 随机抖动
    float noise = random_noise(0.05);
    return constrain(base + dip + noise, 0.05, 0.95);
}
```

### 节律与用户作息的错位

节律基于实时时钟（RTC）或 millis() 累积。如果 Cardputer 没有 RTC 电池，断电后节律重置——这反而自然，因为穆"醒来"后不知道时间，需要重新适应。

不主动同步用户作息。如果用户凌晨 3 点还在，穆确实困了。这不是惩罚，是它的自然状态。

## 交互响应模型

用户的输入（按键、语音）不是直接控制情绪，而是对情绪空间的**扰动**。

### 扰动规则

| 用户行为 | 对愉悦度的影响 | 对唤醒度的影响 | 衰减时间 |
|---|---|---|---|
| 正常说一句话 | +0.2 | +0.3 | ~2min |
| 说了一句有意思的话 | +0.4 | +0.5 | ~5min |
| 反复按键/骚扰 | -0.3 | +0.2 | ~3min |
| 长时间不在（>30min） | -0.1 | -0.2 | 持续 |
| 回来后第一次互动 | +0.3 | +0.4 | ~3min |
| 深夜还在 | 0 | -0.1 | 持续 |

这些值不是精确的——它们会乘以 0.5-1.5 的随机系数。

### 关键设计：间接性

用户不应该能感知到"我做了 X，所以穆变成了 Y"。变化必须足够慢、足够模糊，使得因果关系不明显。

"你回来之后穆好像精神了一点"——这是好的。
"你按了按钮所以穆从无聊变成好奇"——这是坏的。

实现方式：
- 扰动不是瞬间施加的，而是以每帧微量累加的方式施加
- 每次扰动都有随机缩放
- 多个扰动可以叠加，但总变化有上限

## 情绪 → 表达层映射

情绪状态通过六个维度影响 LAYER 2 的渲染：

| 表达维度 | 低值 | 高值 | 受什么驱动 |
|---|---|---|---|
| **眨眼频率** | 困时：每 8-15 秒 | 活跃时：每 2-5 秒 | 能量 |
| **呼吸幅度** | 困时：微弱 | 活跃时：明显 | 能量 |
| **呼吸速度** | 困时：慢 | 好奇时：快 | 唤醒度 |
| **目光移动频率** | 无聊时：几乎不看 | 好奇时：频繁 | 唤醒度 + 愉悦度 |
| **嘴角状态** | 微愠时：略微向下 | 快乐时：略微向上 | 愉悦度 |
| **叹气频率** | 无聊/微愠时：频繁 | 安适时：极少 | 愉悦度（反向） |
| **微动频率** | 困时：几乎不动 | 活跃时：频繁微动 | 能量 |
| **声音频率** | 困时：极少 | 活跃时：偶有轻响 | 能量 |
| **屏幕亮度** | 困时：极暗 | 活跃时：正常 | 能量 |

映射不是离散切换，是连续插值：

```c
// 示例：眨眼间隔
float blink_interval() {
    float base = map(energy, 0, 1, 8000, 2500);  // 8s→2.5s
    float noise = random(-500, 500);
    return base + noise;
}

// 示例：嘴角
int mouth_curve() {
    // valence -1~+1 映射到嘴角 -3~+3 像素
    return (int)(valence * 3);
}
```

## 需要修改的现有代码

Phase 1 的 `mu.ino` 需要以下改动：

### 新增模块

1. **mood.h / mood.cpp**（或在 .ino 中作为一个 section）
   - `mood_tick(dt)` — 每帧更新情绪三元组
   - `rhythm_energy(hour, minute)` — 节律能量计算
   - 交互扰动函数 `mood_disturb(valence_delta, arousal_delta)`

2. **RTC 读取**（可选）
   - 如果有 RTC 电池：读取真实时间
   - 如果没有：用 millis() 累积模拟日内循环（断电重置）

### 修改现有代码

1. **drawFace()** — 从情绪状态计算表达参数
   - 眨眼间隔不再固定，而是 `blink_interval(energy)`
   - 眼睛大小/位置受情绪影响
   - 嘴巴形状受愉悦度影响
   - 呼吸幅度/速度受能量和唤醒度影响

2. **调度器** — 所有事件间隔都基于情绪状态
   - `schedBlink()` → 使用 `blink_interval()`
   - `schedDrift()` → 目标偏移量和频率受唤醒度影响
   - `schedSigh()` → 频率受愉悦度反向影响
   - `schedTwitch()` → 频率受能量影响
   - `schedAmb()` → 频率受能量影响

3. **屏幕亮度** — 能量低时变暗

### 不变的部分

- AnimChannel 结构和动画插值逻辑不变
- 呼吸振荡器仍然独立运行，但参数受情绪影响
- 声音系统不变，但触发频率受情绪影响

## 观测验证

实现后，可以通过以下方式验证情绪系统是否工作：

1. **静置观察 10 分钟**：穆应该在能量曲线上自然移动，表情缓慢变化
2. **连续互动 5 次**：穆应该在几次后变得不那么积极（唤醒度先升后降，愉悦度可能微降）
3. **离开 30 分钟后回来**：穆应该比离开前更安静，第一次互动后逐渐恢复
4. **深夜观察**（如果等得到）：穆应该明显更安静、动作更少、更暗

**不建议**：在屏幕上显示当前情绪值来"调试"。如果需要调试，用串口输出到电脑。穆的屏幕上永远只有脸。

## 不做的事

| 不做 | 原因 |
|---|---|
| 离散状态机（"当前是好奇"） | 太机械，不自然 |
| 用户可以看到的情绪指示器 | 破坏"只能被感知"的设计 |
| 基于用户行为的精确因果 | 用户会"学会"操控穆的情绪 |
| 情绪影响 LLM 回复内容（Phase 4） | 这个以后再考虑，Phase 2 只影响表达层 |
| 复杂的多维情绪模型 | 两个轴 + 能量足够了，更多维度不会更自然 |
